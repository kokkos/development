  - Christian wants it
- MDRangePolicy next step #8629
- Look at register usage in #9133 Restore cuda_parallel_launch_constant_memory
  - grid_constant not quite the same as lauching from constant memory although it should be (on the compiler level)
  - different launch mechanims
    - most straightforward: local (limited size)
    - global memory: copy to global memory before launching, we need to have a specific allocation to manage
    - constant memory: can't launch multiple kernels at once, non-coherent, read-only, higher launch overhead, faster to run sometimes
    - \_\_constant_memory\_\_ should use constant memory implicitly combining benefits of local and constant launch, benefits gets lost for complex kernels resilting in higher register usage

  - We want to be the C++ framework that also supports more advanced use cases
