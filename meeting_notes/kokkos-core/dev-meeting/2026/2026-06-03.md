### 5.2
#### fix deduction for template parameter
 - Deducting return type fails for `parallel_reduce` when functor is templated (can't deduce type)
 - Daniel and Christian don't agree with Yuuichi's proposed fix and proposed something different
 - But agree to support templated functors
 - Using void like Yuuichi propose would make deducing the correct block size/grid size impossible
 - We should allow to pass the value type so that we don't hit the functor analysis part

 - Part of what Yuuichi is proposing is to improve the error message, Christian don't want to includ it either because it risk breaking an already fragile code. He doesn't think it would be worth taking this risk only to improve the error message

#### Only run githunb actions for push events to kokkos/kokkos repo
 - We don't pay for the Github CI but we can see "how much it would cost if we paid".
 - This PR changes would only run CI for PRs, not for pushes on forks
