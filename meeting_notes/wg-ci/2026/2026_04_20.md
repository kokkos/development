* [Googletest update](https://github.com/kokkos/kokkos/pull/8593) - some issues with Windows builds, we need some assistance from someone with Windows experience. Christian may have time to help
* Trevis reported we do not have SIMD fallback testing when ranges not available (following updates in [PR 9065](https://github.com/kokkos/kokkos/pull/9065)); we need to identify or create a new job with a suitable environment to add coverage
* Multigpu testing [PR update](https://github.com/kokkos/kokkos/pull/8995) - Paul mentioned concerns about whether multigpu tests are actually run in CI; Nathan confirmed the SNL-CI Cuda and Hip jobs run on nodes with multiple gpus available and accessible through the container; we need to check if additional e.g. env variables (KOKKOS_NUM_DEVICES > 1?) are needed in order to satisfy conditions for the test to run
* Improving reproducibility of CI failures
  * Nathan revisiting python scripting to automatically extract job, image/runner, and cmake config from CI yml files to a json file or "digest" - idea is to have easy way to extract key reproducer info that can be used with a pr reproducer type script, as well as quickly search for config info etc.
  * Paul suggested revisiting cmake-presets as a way to store and easily load configurations; need to check if this would work with the existing workflows, e.g. hpsf-ci
  * We have Dockerfiles for various configurations available, but need to register images at least for the most expensive images to pull+build. Nathan will look at registering the SNL-CI images at https://github.com/kokkos/ci-containers/tree/main
  * How should we handle for example jobs running on bare metal rather than in a container? Difficulties reproducing environment exactly
